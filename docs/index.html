3 · Normalization & feature selection

Counts are normalized to a common library size, log-transformed, and we keep the most variable genes — the biological signal that separates cell types from technical noise.

Show code 
sc.pp.normalize_total(adata, target_sum=1e4)
sc.pp.log1p(adata)
adata.raw = adata  # keep full log-normalized matrix for marker plotting/annotation

sc.pp.highly_variable_genes(adata, min_mean=0.0125, max_mean=3, min_disp=0.5)
n_hvg = int(adata.var.highly_variable.sum())
adata = adata[:, adata.var.highly_variable].copy()
sc.pp.regress_out(adata, ["total_counts", "pct_counts_mt"])
sc.pp.scale(adata, max_value=10)
print(f"Selected {n_hvg} highly variable genes for clustering.")
Selected 1838 highly variable genes for clustering.

6 · Cell-type annotation (data-driven)

Rather than hand-label cluster numbers (which is fragile), we score each cell against canonical PBMC marker panels and assign every cluster the best-matching lineage by mean score. This is reproducible and defensible.

Show code 
marker_sets = {
    "CD4 T":        ["IL7R", "CD3D", "CD3E", "CCR7"],
    "CD8 T":        ["CD8A", "CD8B", "CD3D"],
    "B":            ["MS4A1", "CD79A", "CD79B"],
    "NK":           ["GNLY", "NKG7", "KLRD1"],
    "CD14+ Mono":   ["CD14", "LYZ", "S100A8"],
    "FCGR3A+ Mono": ["FCGR3A", "MS4A7"],
    "Dendritic":    ["FCER1A", "CST3"],
    "Megakaryocyte":["PPBP", "PF4"],
}
for ct, genes in marker_sets.items():
    present = [g for g in genes if g in adata.raw.var_names]
    sc.tl.score_genes(adata, present, score_name=f"score_{ct}", use_raw=True)

score_cols = [f"score_{ct}" for ct in marker_sets]
cluster_scores = adata.obs.groupby("leiden", observed=True)[score_cols].mean()
assignment = cluster_scores.idxmax(axis=1).str.replace("score_", "", regex=False)
adata.obs["cell_type"] = adata.obs["leiden"].map(assignment).astype("category")

summary = (adata.obs["cell_type"].value_counts()
           .rename_axis("cell_type").reset_index(name="n_cells"))
summary["pct"] = (100 * summary["n_cells"] / summary["n_cells"].sum()).round(1)
summary.to_csv("results/celltype_composition.csv", index=False)

8 · Interpretation & caveats

NoteWhat this means — and what I would not conclude

The pipeline cleanly resolves seven canonical PBMC populations — CD4 T cells, both monocyte subsets (classical CD14+ and non-classical FCGR3A+), B cells, NK cells, dendritic cells, and a small megakaryocyte/platelet cluster — each supported by textbook markers in the dot-plot. CD8 T cells sit adjacent to CD4 T cells and are not split at this depth — an honest limitation of a shallow 3k dataset, flagged rather than papered over. That internal consistency is the real quality signal: the clusters are biologically interpretable, not just statistically distinct.

Caveats a careful analyst flags: (1) cluster boundaries depend on the Leiden resolution — 1.0 here separates seven lineages but leaves CD8 T adjacent to CD4 T; lowering it further merges the non-classical monocytes into NK, while raising it can split CD8 out at the cost of stability and spurious clusters — so resolution should be chosen against the biological question, not by default. (2) Marker-score annotation labels a cluster by its best match — rare or transitional states can be absorbed into a neighbour; reference-based tools (e.g. CellTypist) are the next step when labels must be defensible. (3) This is a single healthy donor: no batch, no condition, so no differential abundance/expression claims are made here. (4) Doublets were handled only via the upper gene-count filter; a dedicated detector (Scrublet) is warranted on real multi-sample data.

What I’d do with your data: confirm the biological question first (atlas vs comparison), tune QC/resolution to it, add doublet detection and — for multi-sample designs — integration (Harmony/scVI) before any cross-condition comparison.
